# รายการ SQL ที่ต้องรันเพิ่ม (หลัง Merge Branch)

จากการตรวจสอบความแตกต่างของโค้ดใน Branch ต่างๆ ที่ถูก Merge เข้าสู่ `main` พบว่ามีการอัปเดตฐานข้อมูลที่สำคัญในส่วนของ **Codegen v2** (ระบบสร้างโค้ดอัตโนมัติ) ดังนี้ครับ:

## 1. ไฟล์ SQL หลักที่ต้องรัน
คุณต้องนำโค้ดจากไฟล์นี้ไปรันใน Supabase SQL Editor:
- **ไฟล์:** `supabase/migration_codegen_v2.sql`

## 2. สรุปการเปลี่ยนแปลงใน SQL นี้
| หัวข้อ | รายละเอียดการแก้ไข |
|---|---|
| **สถานะออเดอร์** | เพิ่มสถานะ `review` ในตาราง `codegen_orders` เพื่อใช้ในขั้นตอนตรวจสอบโค้ดก่อนส่งมอบ |
| **Storage Bucket** | สร้างหรืออัปเดต Bucket ชื่อ `codegen` เพื่อใช้เก็บไฟล์ ZIP และรูปภาพสลิป |
| **Storage Policies** | กำหนดสิทธิ์ (RLS) ให้ Admin สามารถอัปโหลด/แก้ไขไฟล์ได้ และผู้ใช้ทั่วไปสามารถดาวน์โหลดไฟล์ของตัวเองได้เท่านั้น |
| **Table Security** | ปรับปรุง RLS ของตาราง `codegen_orders` ให้รัดกุมขึ้น (ผู้ใช้เห็นเฉพาะออเดอร์ตัวเอง, Admin เห็นทั้งหมด) |

## 3. วิธีการดำเนินการ
1. เข้าไปที่ [Supabase Dashboard](https://supabase.com/dashboard) ของโปรเจกต์คุณ
2. ไปที่เมนู **SQL Editor**
3. คัดลอกเนื้อหาจากไฟล์ `supabase/migration_codegen_v2.sql` ในโปรเจกต์ไปวางและกด **Run**
4. ตรวจสอบผลลัพธ์ว่าสำเร็จครบทุกส่วนหรือไม่

---
*หมายเหตุ: หากคุณเคยรัน SQL บางส่วนไปแล้ว ระบบจะข้ามส่วนที่ซ้ำซ้อน (เช่น การสร้าง Bucket) ให้โดยอัตโนมัติผ่านคำสั่ง `ON CONFLICT` และ `DROP CONSTRAINT IF EXISTS`*
